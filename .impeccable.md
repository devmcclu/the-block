## Design Context

### Users
Professional vehicle buyers making high-value purchasing decisions in a wholesale auction environment. They need to browse inventory efficiently, evaluate vehicles quickly, and place bids with confidence. Trust, clarity, and credibility are paramount — these users are spending serious money and need the interface to feel reliable and authoritative.

### Brand Personality
**Bold, Modern, Confident.** "The Block" presents itself as a contemporary auction platform that commands attention without being flashy. The tone is professional but not sterile — it has presence and conviction. Think strong typographic hierarchy, decisive use of space, and an interface that feels like it was built by people who understand the industry.

### Aesthetic Direction
- **Visual tone:** Clean and data-forward with bold structural elements. Professional density without feeling cluttered. The interface should prioritize scannability and fast decision-making.
- **Reference:** OPENLANE / Manheim — industry wholesale platforms that are data-dense, utilitarian, and professional. Lean into the functional strengths of these platforms while bringing a more modern, polished execution.
- **Anti-reference:** Avoid consumer-facing car shopping sites that over-emphasize lifestyle imagery and emotional selling. This is a tool for professionals, not a showroom brochure.
- **Theme:** Both light and dark mode, with the neutral OKLCH palette already in place. The monochromatic foundation is a strength — let vehicle images provide the color.
- **Typography:** Geist — a clean, modern sans-serif that supports the bold/confident personality at heavier weights while remaining highly legible for data-dense views.

### Design Principles
1. **Clarity over decoration** — Every element should earn its place. Favor clear data presentation, strong hierarchy, and purposeful whitespace over ornamental design.
2. **Confidence through consistency** — Uniform spacing, predictable interactions, and a cohesive component system build trust. Users should never feel uncertain about what an element does.
3. **Density done right** — Professional users want information density, but it must be structured. Use typographic scale, subtle dividers, and grouped sections to make dense layouts scannable.
4. **Bold restraint** — The design should feel strong and modern (bold typography, decisive layout choices) but never loud. Restrained use of color means that when something is highlighted, it matters.
5. **Speed of comprehension** — Optimize for fast scanning and quick decisions. Badges, status indicators, and numerical data should be immediately parseable at a glance.

### Accessibility
- WCAG AA compliance for contrast ratios, keyboard navigation, and screen reader support
- Focus indicators on all interactive elements
- Proper semantic HTML and ARIA attributes
- Respect `prefers-reduced-motion` for users who need it

### Technical Foundation
- Tailwind CSS v4 with OKLCH color tokens (CSS custom properties)
- shadcn-vue (new-york style, reka-ui base, Geist font)
- Neutral base color palette — monochromatic by design
- Icons via @iconify/vue with hugeicons library
- Dark mode via @vueuse/core useColorMode + Tailwind dark: prefix
